## 三、已知问题

### 1. 刷新失败曾返回 500(已修复 → 400)

`response.ErrorFrom` 对非 `ApplicationError` 的普通 error 默认映射 500。`RefreshProvider` 过盾/采集失败返回普通 error,导致 handler 返回 500 internal error。
**修复**(commit `c52bbf48`):刷新失败(非 RefreshInProgress)→ `BadRequest`(400)+ 友好信息;`NotFound` 仍走 404。前端从详情可见持久化的 `last_error`。

### 2. CloakBrowser 在 Docker Desktop for Mac 下 TLS 握手失败(外部依赖/环境)

CloakBrowser 的 patched Chromium 打开 HTTPS 时 `handshake failed; net_error -100`(arm64/amd64 均复现),而容器内系统 `curl` 同站点 200。判断为 **Docker-Mac NAT + MTU 导致 Chromium TLS 大包被丢弃**的经典问题,非本模块代码问题。
本模块集成链路(chromedp 连 CloakBrowser、传 fingerprint-seed、导航、cf_clearance 轮询、超时优雅返回 400)**已验证正常**。规避见 [本地测试与部署.md §三](本地测试与部署.md)(低 MTU 网络);Linux 生产通常无此问题。
> 补充:为「采集设置」配一个干净(住宅)代理后,**HTTP 采集常可直接通过 CF**(login 成功即拿到数据),浏览器过盾仅在站点强制 challenge 时才需要——本环境问题影响面因此进一步缩小。

### 3. `read-only /app` 日志降级(无害)

`make dev-backend` 在宿主机跑时,默认日志文件路径 `/app/data/logs` 只读,降级到 stdout(`LOG_OUTPUT_TO_STDOUT=true` 兜底)。仅 WARN,不影响功能。如需消除见故障排查表。

### 4. 前端 16 个 pre-existing 测试失败(非本模块引入)

`pnpm test:run` 有 16 个失败,均为既有回归(`AccountUsageCell`/`*DistributionChart`/`EmailVerifyView`/`UsageView`/`UsageTable`,chart canvas mock / image billing 断言等),本模块未触碰这些文件;`AppSidebar`/`upstreamValidators`/i18n 守卫测试均通过。

## 四、后续优化建议(非阻塞)

- `UpstreamMonitorRunner` 优雅停机:当前不注册 shutdown cleanup(R1.7 决策,靠 5min stale 锁兜底),如需进程退出时优雅 drain in-flight 刷新,可将刷新 goroutine 纳入 runner 的 WaitGroup。
- BrowserSolver 截图正则耦合中文「诊断截图:」前缀(前端解析失败截图文件名用),若错误消息多语言化需同步。
- sub2api 适配器 `profile.Code != 0` 但 HTTP 200 时静默 partial(本仓库 profile 认证失败走 HTTP 401,实际不触发);可加显式上抛增强健壮性。

### 🔧 端到端抓出的真实缺陷(已修复)
**漏写 SQL 迁移**:本仓库运行期建表靠手写 `migrations/*.sql`,`make generate` **不**生成迁移。spec / 计划 / Task 1 均只跑了 `make generate`,**漏了 `151_upstream_usage_collection.sql`** → 旧后端无新表/列(`/usage` 404、`recharge_ratio` 不回显)。e2e 发现后补齐并提交(`7b13c8b5`)。**教训**:凡新增 ent 实体/字段,必须同步写编号 SQL 迁移(见 `架构设计.md §一` 警示)。

### 已知限制(消耗采集)
- newapi **分组改名无法归一**(日志 `group` 仅名、无 id)→ 当新分组;密钥改名可归一(有 token_id)。
- sub2api **无分组消耗维度**(用户凭证不暴露)→ breakdown `supported:false`。
- **退款归位可能不全**:newapi 退款(type=6)若缺 token_id/group 只冲减 total,致 key/group 之和略高于 total;total 始终准。
- **回填受上游日志保留期限制**(newapi `LOG_RETENTION`);拿不齐则"历史"以 `backfill_oldest_day` 为准标注。
- **冻结区(早于可变窗 2 天)不接受迟到修正**——权衡换"免被上游清日志误删历史";如需更大纠正窗可调 `K`(`K` 越大被误删风险越高)。
- **采集翻页复用 `upstreamSnapshotTimeout`(整次 60s)**,对日志页数多的繁忙上游(叠加慢代理)可能超时 → 当轮不写、下轮重试;后续可给采集单独配更长超时或缩单轮页数上限(phase 1 未做)。
